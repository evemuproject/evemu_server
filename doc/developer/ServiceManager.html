Services manager

After we initialize the service manager.
//make the item factory
ItemFactory item_factory( sEntityList );
sLog.Log("server init", "starting item factory");

//now, the service manager...
PyServiceMgr services( 888444, sEntityList, item_factory );
sLog.Log("server init", "starting service manager");
Then we can register a service. In this case the inventory broker service.
    services.RegisterService(new InvBrokerService(&services));
  RegisterService simply adds the created object to the PyServiceMgr m_services list. InventoryBrokerService constructor registers the name "invbroker" with the service manager. Thats all untill the servece gets a request.
  When a client wants an inventory item it will send a MachoBindObject reqest with the method service name invbroker.
  When the server recieves MachoBindObject call in Client::handle_CallReq it will check to see if there is a service name. If there is not a service name it will try to locate the previously bound object, if there is a service name it will look up the service. In our example the servie name (packet->dest.service) is "invbroker" so the InvBrokerService object is retrieved.
  A call is then made to the service object. In this case we have a InvBrokerService which is derivecd from PyService but does not override the Call function, so fuction PyService::Call is called. The PyService checks to see what type of request if being made, in this case a bind object request (MachoBindObject). so the Handle_MachoBindObject function is invoked.
  The PyService::Handle_MachoBindObject then creates a bound object to return to the client, in this case a InvBrokerBound object. The bound objects Call function is then called to handle the sqecific request.
  This is not overridden so it's base class is called (PyBoundObject::Call). This simply ouputs a debug message about the call and passes it the the base class PyCallable::Call
  The PyCallable then dispatches the request to an appropriate handler. In our case we have a method name of "GetInventoryFromId". The method was registered with. 
    PyCallable_DECL_CALL(GetInventoryFromId)
  The registered call is invoked Handle_GetInventoryFromId. This loads the specified inventory item and creates an InventoyBound object to contain it, that is bound to the client and returned.
  The InvBrokerBound and InventoryBound objects are now bound to the client and available to be used in subsiquent calls from the client. Such as to add or move items from one location to another.
