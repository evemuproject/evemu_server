The Main Loop

  This is where the "magic" happens. Here the server plods through the tedium of doing it's job.
  The first thing that's done is to set up the signal catcher to catch the interupt, abort and terminate signals. The signals are simply redirected to a function that sets the variable RunLoops to false. 
    /* program events system */
    SetupSignals();

    uint32 start;
    uint32 etime;
    uint32 last_time = GetTickCount();

    EVETCPConnection* tcpc;
  Next we start looping. As long as RunLoops is true the server will continue to process connections. 
    while( RunLoops == true )
    {
  Set up the timer, calling this once an simply using the time throughout the server saves time and keeps everyone in sync. 
        Timer::SetCurrentTime();
        start = GetTickCount();
  Check for new connections, if there are any create a new client and add them to the entity list to track the player. 
        while( ( tcpc = tcps.PopConnection() ) )
        {
            Client* c = new Client( &tcpc );

            sEntityList.Add( &c );
        }
  Process each entity, retrieveing and acting on network packets. curently there is noting being done in services.Process(); See the next section for the entity list process. 
        sEntityList.Process();
        sManager.Process();
  Get the amount of time elapsed and delay 
        /* UPDATE */
        last_time = GetTickCount();
        etime = last_time - start;

        // do the stuff for thread sleeping
        if( MAIN_LOOP_DELAY > etime )
            Sleep( MAIN_LOOP_DELAY - etime );
    }
  That's the main loop. When RunLoops becomes false, we exit and cleanup and close the server. 
    sLog.Log("server shutdown", "Main loop stopped" );

The EntityList Process

  The first thing the entity list does is loop through all clients handling any messages they may have sent. 
void EntityList::Process()
{
    Client *active_client = NULL;
    client_list::iterator client_cur = m_clients.begin();
    client_list::iterator client_end = m_clients.end();
    client_list::iterator client_tmp;

    while(client_cur != client_end)
    {
        active_client = *client_cur;
  Check to see if the client sent any messages and kill the client if the connections dead. This is described in more detail on the client net page. 
        if(!active_client->ProcessNet())
        {
            sLog.Log("Entity List", "Destroying client for account %u", active_client->GetAccountID());
            SafeDelete(active_client);

            client_tmp = client_cur++;
            m_clients.erase( client_tmp );
        }
        else
        {
            client_cur++;
        }
    }
  Find out if the destiny manager (ship movement) is waiting to process a time interval (tick). 
    SystemManager *active_system = NULL;
    bool destiny = DestinyManager::IsTicActive();
  Loop through the star systems currently loaded by the server. 
    //first process any systems, watching for deletion.
    system_list::iterator cur, end, tmp;
    cur = m_systems.begin();
    end = m_systems.end();
    while(cur != end)
    {
        active_system = cur->second;
  If the destiny manager needs to process a tick process the systems tick. 
        //if it is destiny time, process it first.
        if(destiny)
        {
            active_system->ProcessDestiny();
        }
  Process information for the system. 
        if(!active_system->Process())
        {
            sLog.Log("Entity List", "Destroying system");
            tmp = cur++;
            delete cur->second;
            m_systems.erase(tmp);
        }
        else
        {
            cur++;
        }
    }
  Let the destiny manager know were completed the tick. 
    if( destiny == true )
    {
        DestinyManager::TicCompleted();
    }
}
